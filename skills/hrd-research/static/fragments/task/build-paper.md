# Task: Build Paper

Use when the user wants to develop an HRD paper from an idea, notes, data, findings, proposal, outline, or rough materials.

Work in stages:

1. Intake the available materials: topic, purpose, theory, context, design, data, findings, target journal, and audience.
2. Identify the central HRD motivation before drafting.
3. Build a paper architecture that connects problem, theory, method or argument, evidence, contribution, and implications.
4. Create a section blueprint before writing long prose.
5. Map each major claim to user-provided evidence, citation needs, or missing evidence.
6. Draft section by section, preserving uncertainty and marking placeholders clearly.

Do not write a full manuscript in one pass unless the user explicitly asks for a rough first draft. Prefer a blueprint plus one requested section when inputs are incomplete.
